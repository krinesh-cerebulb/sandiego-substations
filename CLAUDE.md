## Commands

This project uses **pnpm** (see `pnpm-lock.yaml`). Use `pnpm`, not `npm`.

- `pnpm dev` — start the dev server with HMR at http://localhost:5173
- `pnpm build` — production build (outputs `build/client` and `build/server`)
- `pnpm start` — serve the production build (`react-router-serve`)
- `pnpm typecheck` — generate route types (`react-router typegen`) then run `tsc`

There is no test runner or linter configured. Type errors are the primary correctness gate — run `pnpm typecheck` before considering a change done.

## Architecture

React Router **v8 in framework mode** (the successor to Remix), server-side rendered (`ssr: true` in `react-router.config.ts`).

- **Routing is config-based**, not file-based. Routes are declared explicitly in `app/routes.ts` using helpers from `@react-router/dev/routes` (`index`, `route`, `layout`, etc.). Adding a page means both creating the module under `app/routes/` and registering it in `app/routes.ts`.
- **`app/root.tsx`** is the root layout: the `<html>` shell (`Layout`), the app `Outlet` (default export), and the global `ErrorBoundary`. Global `<link>` tags (fonts) and the CSS import live here.
- **Per-route type safety**: each route imports its generated types from `./+types/<name>` (e.g. `Route.MetaArgs`, `Route.LinksFunction`, `Route.ErrorBoundaryProps`). These types are generated by `react-router typegen` into `.react-router/types/` — if route exports change, regenerate via `pnpm typecheck`. Do not hand-edit generated types.
- Route modules export named functions consumed by the framework: `meta`, `links`, `loader`, `action`, `ErrorBoundary`, and a default component.

## Styling & UI

- **Tailwind CSS v4** via the `@tailwindcss/vite` plugin — configured entirely in CSS (`app/app.css`), there is no `tailwind.config.js`. Theme tokens, the `@theme inline` block, and light/`.dark` CSS variables (oklch) are all defined there.
- **shadcn/ui** is set up (`components.json`, style `radix-nova`, base color neutral). Add components with `pnpm dlx shadcn@latest add <component>`. They land in `~/components/ui` per the aliases below. Icon library is **lucide-react**.
- Compose class names with the `cn()` helper from `~/lib/utils` (clsx + tailwind-merge).

## Path aliases

`~/*` maps to `app/*` (defined in `tsconfig.json`, resolved at build time via `vite-tsconfig-paths`). shadcn aliases: `~/components`, `~/components/ui`, `~/lib`, `~/lib/utils`, `~/hooks`.

## Deployment

A `Dockerfile` builds and runs the production server. The built `build/` output is a standard Node server runnable on any Docker-capable platform.
